# 系统定位

Clover 不是单一工具包，而是一套把运行时协议、边界适配、静态约束和工程习惯一起设计的 TypeScript SDK 体系。

它的目标不是提供“更多工具函数”，而是把一组明确的工程判断固化下来：

- 核心域优先依赖静态类型，而不是运行时兜底
- 空值和错误都必须显式建模
- 运行时表面尽量薄，避免额外包装层
- 数据结构、调用点和数值域尽量朝 V8 友好的稳定形态收敛
- 规范不只写在文档里，还要进入 ESLint 和 tsconfig

它要覆盖的不是某一个包，而是整条开发链路：

- 最小协议核心
- 运行时标准库
- Zod 边界适配
- CLI 边界适配
- HTTP 请求响应边界
- TypeScript 预设
- ESLint 规则和配置
- 基准与验证脚本

它明确不想变成以下形态：

- 一个没有边界的 utils 大杂烩
- 以 class 和异常为中心的领域框架
- 把 Zod schema 直接带入核心业务模型的运行时系统
- 只靠 code review 口头约束、没有工具落地的“风格偏好”

从系统视角看，Clover 的价值在于把“代码应该长什么样”这件事做成可复用的工程产品。
