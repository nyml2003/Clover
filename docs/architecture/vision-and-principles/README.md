# 愿景与原则

这一组文档解释 Clover 的系统定位、设计主张和边界纪律。

阅读建议：

1. 先看 `system-positioning.md`，理解它到底想解决什么问题
2. 再看 `system-map.md`，理解整套系统现在和以后分别会长成什么样
3. 再看 `automation-base.md`，理解自动化基座不是 task runner，而是按配置和规约求解执行图的系统
4. 再看 `design-principles.md`，理解核心设计公理
5. 最后看 `architectural-boundaries.md`，理解哪些能力应该放在哪里

这一部分只回答“为什么这样设计”，不讨论当前实现缺口。
